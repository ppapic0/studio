# Google Sheets 운영보드 리디자인 실행 가이드

이 문서는 `트랙 관리형 스터디센터 운영보드`를 직접 수정하지 않고, 복사본에 새 운영 허브를 생성하는 Apps Script 실행 절차입니다.

## 생성되는 것

- 원본 파일과 같은 Drive 폴더에 `트랙 운영보드 리디자인_YYYYMMDD` 복사본 생성
- 복사본 앞쪽에 새 탭 8개 추가
  - `00_운영 허브`
  - `01_업무 큐`
  - `02_인수인계`
  - `03_운영 체크`
  - `04_상담·연락`
  - `05_결제·재등록 체크`
  - `06_운영 리포트`
  - `99_설정`
- 기존 원본 탭은 복사본 뒤쪽에 유지하고, 회색 탭 색상과 경고 보호를 적용

## 실행 방법

1. 원본 Google Sheet를 엽니다.
2. `확장 프로그램 > Apps Script`로 이동합니다.
3. [`operations-board-redesign.gs`](../scripts/google-sheets/operations-board-redesign.gs)의 전체 내용을 Apps Script 편집기에 붙여넣습니다.
4. 저장 후 함수 목록에서 `createRedesignedOperationsBoard`를 선택합니다.
5. `실행`을 누르고 Google 권한 승인을 완료합니다.
6. 완료 알림에 표시되는 URL을 열어 복사본을 확인합니다.

## 확인 체크리스트

- 원본 파일의 탭, 데이터, 서식이 바뀌지 않았는지 확인합니다.
- 복사본 앞쪽에 새 탭 8개가 생성되었는지 확인합니다.
- `00_운영 허브`의 `운영일`을 바꾸면 업무/근무자/특이사항 요약이 갱신되는지 확인합니다.
- `01_업무 큐`에서 학생명을 입력하면 학부모/학생 연락처가 조회되는지 확인합니다.
- `상태`, `우선순위`, `담당자` 드롭다운과 상태별 조건부서식이 동작하는지 확인합니다.
- 기존 DB 탭은 삭제되거나 이름이 바뀌지 않았는지 확인합니다.

## 운영 메모

- 조교 선생님은 주로 `00_운영 허브`, `01_업무 큐`, `02_인수인계`, `03_운영 체크`를 사용하면 됩니다.
- 관리자 확인은 `06_운영 리포트`와 기존 DB 탭을 함께 보면 됩니다.
- 연락처는 업무 처리를 위해 `01_업무 큐`, `04_상담·연락`, `05_결제·재등록 체크`에 노출됩니다.
